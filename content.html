Root Cause Analysis

Primary Intermittent Outbound Network Failure

The Hostinger VPS experienced transient failures reaching api.telegram.org (149.154.166.110). The Node.js undici fetch client threw repeated TypeError: fetch failed errors, causing both sendMessage and sendChatAction calls to fail.

This explains the full symptom: the dashboard (local WebSocket, loopback) was instant, while Telegram delivery (outbound HTTPS) was blocked.

Contributing Long Polling Mode

Telegram is configured in polling mode rather than webhook mode. While not the cause of failed deliveries, polling adds baseline latency (the gateway periodically asks Telegram for updates rather than receiving instant pushes).
